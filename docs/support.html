Getting Started

Follow the Quick Start guide in the GitHub README to set up CodeSentinel in your Slack workspace. You'll need a Slack Developer Program sandbox, a Bolt.js app created from the included manifest, and an LLM API key.

Frequently Asked Questions

How do I connect CodeSentinel to my repository?

Mention CodeSentinel in a Slack message with a GitHub or GitLab URL, e.g. @CodeSentinel run full health scan on https://github.com/org/repo. You can also use the suggested prompts in the assistant view without specifying a repository.

Which LLM providers are supported?

CodeSentinel supports Deepseek (default), OpenAI, and Anthropic. Set the LLM_PROVIDER environment variable and the corresponding API key in your .env file.

Can I use CodeSentinel outside of Slack?

Yes. CodeSentinel ships a standalone MCP server with 6 analysis tools. You can connect it to Claude Desktop, Cursor, Windsurf, or any MCP-compatible client by adding the server configuration to your MCP settings file.

The app shows "Analyzing..." but never responds

Check that your SLACK_APP_TOKEN is set correctly (starts with xapp-) and that Socket Mode is enabled in your Slack app settings. Also verify your LLM API key is valid and has available credits.

Report a Bug

Open an issue on GitHub Issues. Include your Node.js version, Slack app configuration, and any relevant error logs from the console.

Feature Requests

We welcome ideas for new analyzers, integrations, or improvements. Open a GitHub Issue with the enhancement label.
