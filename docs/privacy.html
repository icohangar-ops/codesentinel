Privacy Policy

Last updated: June 13, 2026

Overview

CodeSentinel is an open-source Slack application that analyzes codebase health. This privacy policy explains what data CodeSentinel collects, how it is used, and how it is protected.

Data Collection

CodeSentinel collects and processes the following types of data:

  • Slack Messages: When a user interacts with CodeSentinel in Slack, the app reads the user's message to determine the analysis intent (e.g., "scan for dead code"). Messages are processed in real-time and are not stored persistently.
  • Repository URLs: If a user provides a GitHub or GitLab URL, CodeSentinel fetches public repository metadata (name, default branch, language) via their respective APIs. No code content is downloaded or stored.
  • Analysis Results: Analysis results are returned as Slack messages in the conversation thread. Results are generated using built-in demo analyzers and are not stored on any server.

LLM API Usage

CodeSentinel can optionally use an external LLM API (Deepseek, OpenAI, or Anthropic) to generate AI summaries of analysis results. When enabled, analysis findings are sent to the configured LLM provider for summarization. The LLM provider's own privacy policy governs how that data is handled. You can disable AI summaries by omitting the LLM API key from your configuration.

No Persistent Storage

CodeSentinel does not maintain a database, log file, or any persistent storage of user data. All processing happens in-memory during the request-response cycle. Analysis results exist only as Slack messages in the user's conversation thread, subject to Slack's own data retention policies.

Third-Party Services

CodeSentinel may interact with the following third-party services:

  • Slack: Message delivery and Block Kit UI rendering — governed by Slack's Privacy Policy.
  • GitHub / GitLab: Public repository metadata API — governed by their respective privacy policies.
  • LLM Providers: Optional AI summarization — governed by the provider's privacy policy.

Data Sharing

CodeSentinel does not sell, share, or transfer any user data to third parties for marketing or advertising purposes. Data is only shared with the services listed above as necessary to provide CodeSentinel's functionality.

Security

CodeSentinel runs on your own infrastructure (or locally). Slack communication is encrypted via TLS. API keys are stored in environment variables and are never transmitted to any server other than the intended API endpoint. The source code is open-source and available for security review on GitHub.

Slack Scopes

CodeSentinel requests the minimum Slack OAuth scopes necessary to function. The bot token has access to read and write in channels where it is invited. It does not access DMs or channels it has not been added to, except for the direct message channel with the user who invokes it.

Children's Privacy

CodeSentinel is a developer tool intended for use by adults in professional software development environments. It is not directed at children under the age of 13.

Changes to This Policy

We may update this privacy policy from time to time. Changes will be posted on this page with an updated revision date. Continued use of CodeSentinel after changes constitutes acceptance of the updated policy.

Contact

For privacy-related questions or concerns, please open an issue on GitHub Issues.
